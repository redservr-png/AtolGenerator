# 1С УТ 10.3 и COMConnector

Программа подключается к 1С через внешнее COM-соединение `V83.COMConnector`.
Официальное описание: [Внешнее соединение](https://v8.1c.ru/platforma/vneshnee-soedinenie/).

## Что читает программа

Для реализаций используются номер и дата документа, заказ, контактное лицо,
сумма, договор, подразделение, номер чека ККМ, ФП и дата печати чека. Для
Obsidian-кейсов дополнительно читаются комментарий и доступные фискальные поля
ПКО, РКО и оплаты картой.

Номенклатура загружается, когда нужна обычная или исправительная пара чеков.
Для одиночного чека коррекции ФФД 1.05 табличная часть не формируется, но договор,
подразделение и классификация услуги нужны для определения НДС.

## Подключение

Настройки хранятся в `onec_settings.json` рядом с exe:

- сервер `Srvr`;
- имя базы `Ref`;
- пользователь;
- пароль.

COM работает только в Windows. Разрядность приложения должна совпадать с
доступным COM-компонентом 1С. Если `V83.COMConnector` не зарегистрирован:

```powershell
regsvr32 "C:\Program Files\1cv8\<версия>\bin\comcntr.dll"
```

Команду запускают от имени администратора. Для 32-разрядной платформы путь может
находиться в `Program Files (x86)` и следует использовать x86-сборку программы.

Подробности: [документация 1С EDT](https://its.1c.ru/db/content/edtdoc/src/topics/t000419.html).

## Запись обратно в 1С

AtolGenerator не записывает фискальные реквизиты документов напрямую через COM.
После сопоставления формируется CSV Windows-1251 с разделителем `;`. Его загружает
внешняя обработка 1С в интерактивном сеансе пользователя.

Поэтому в истории изменений автором является пользователь, запустивший обработку.
Недоступность реквизита в обычной форме документа не мешает обработке изменить
его программно при наличии прав.

Контракт обработки: [ТЗ импорта](../ТЗ_доработка_обработки_1С_импорт_чеков.md).

## Режимы CSV

- `update_fields` - одиночный чек коррекции: заполнить ФПД, ФД, дату и комментарий;
- `comment_only` - исправительная пара: оставить исходный неправильный чек в
  реквизитах реализации и добавить данные двух новых чеков только в комментарий.

Повторный импорт не должен создавать дубли по ФПД.
