# Сопоставление АТОЛ, ОФД и XML

## Источники

| Источник | Что даёт |
|---|---|
| Локальный `punched_checks.jsonl` | Чеки, успешно отправленные программой через API |
| Журнал АТОЛ CSV | UUID, External Id, операция, статус, ФД, ФПД и дата |
| Отчёт Такском XLSX | Фактический документ ОФД, ФД/ФПД, ККТ, торговая точка и сумма |
| Сгенерированный XML | Номер реализации, операция, External Id и основание |

При запуске программа автоматически пытается открыть последние подходящие
отчёты из рабочих папок, пользовательской папки загрузок и сохранённых путей.

## Сопоставление

1. XML связывается с журналом АТОЛ строго по `ExternalId`.
2. Из строки АТОЛ берутся ФПД, номер ФД и фактическая дата.
3. АТОЛ связывается с ОФД по ФПД.
4. При наличии ОФД дополнительно проверяются ФД и абсолютная сумма.
5. Одинаковая сумма сама по себе не считается уникальным ключом.
6. Дубли по `ExternalId` или ФПД требуют ручной проверки.

Если отчёт ОФД не загружен, строка может быть подготовлена по АТОЛ, но получает
статус `ОФД не загружен`. Это не равно подтверждению ОФД.

## Операции

- одиночный `sell_correction` - режим `update_fields`;
- официальная исправительная пара ФФД 1.05 `sell_refund + sell` - обе строки
  `comment_only`;
- старая пара `sell_refund + sell_correction` читается для обратной совместимости;
- одиночный `sell` или `sell_refund` не экспортируется как коррекция реализации.

## CSV для 1С

Формат: Windows-1251 без BOM, разделитель `;`.

```text
НомерРеализации;ТипЧека;РежимЗаписи;ExternalId;ФПД;НомерФД;ДатаЧека;Комментарий
```

`update_fields` заполняет в реализации ФПД, ФД, дату печати и комментарий.
`comment_only` оставляет реквизиты первоначального неправильного чека без
изменения и дописывает только сведения о новых исправительных чеках.

Пример комментария официальной пары:

```text
17.07.2026 Пробит исправительный чек "Возврат прихода" ФП: 1234567890   ///   17.07.2026 Пробит исправительный чек "Приход" ФП: 9876543210
```

Полный контракт: [ТЗ внешней обработки](../ТЗ_доработка_обработки_1С_импорт_чеков.md).

## Разница АТОЛ и Такском

Журнал АТОЛ содержит документы, прошедшие через АТОЛ Online. Он не является
полным журналом всех физических касс подразделений. Сводный отчёт Такском
охватывает выбранные в кабинете ККТ и торговые точки, но может быть ограничен
фильтрами и максимальным числом строк.

Для случая Obsidian с известным ФПД отсутствие строки в обоих локальных файлах
приводит к поиску карточки чека через WebView2, а не к автоматическому выводу
`чека нет`.
