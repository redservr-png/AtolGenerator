# АТОЛ Онлайн и API v4

Проект работает с API v4, потому что касса использует ФФД 1.05. Официальное
описание: [АТОЛ Онлайн API v4](https://atol.online/api_v4), версия документа 5.23
на дату 17.07.2026.

## Методы API

Базовый адрес: `https://online.atol.ru/possystem/v4/`.

| Действие | Метод |
|---|---|
| Получить токен | `POST getToken` |
| Зарегистрировать документ | `POST {group_code}/{operation}?token={token}` |
| Получить результат | `GET {group_code}/report/{uuid}?token={token}` |

Токен официально действует до 24 часов. Программа кэширует его на 23 часа и
обновляет после ошибки авторизации.

Основные операции: `sell`, `sell_refund`, `buy`, `buy_refund`,
`sell_correction`, `buy_correction`.

## Политика проекта

- API используется для обычного прихода и реального возврата прихода.
- Редкий обычный чек реализации также может быть отправлен через API.
- Коррекции реализаций и все Obsidian-исправления формируются только в XML.
- Кнопка API блокируется как визуально, так и повторной проверкой команды.
- После успешного API-чека UUID, ФД, ФПД и ссылка ОФД записываются в
  `punched_checks.jsonl`.

Официальный API умеет больше, чем разрешено политикой приложения. Ограничение
XML-only принято специально для контролируемой ручной загрузки коррекций и
связанных служебных записок.

## НДС

| Сценарий | Ставка в программе |
|---|---|
| Оплата товара | `vat122` (22/122) |
| Реализация товара | `vat22` |
| Агентская услуга Страхова | `vat5` для реализации, расчётная ставка для оплаты |
| Другие настроенные агентские услуги | `none` |
| `Услуга по доставке (Россия)` интернет-магазина | собственная услуга, без агента, `vat22` |

Поддержка `vat22` и `vat122` добавлена в API v4 документом 5.22. Если услуга не
определилась, коррекционный XML не формируется, но корректные соседние строки не
блокируются.

## Агент и поставщик

Для агентской услуги в позиции обычного чека передаются:

- `agent_info.type = commission_agent`;
- `supplier_info.name`;
- `supplier_info.inn`;
- `supplier_info.phones`.

Агент выбирается по виду услуги и подразделению 1С. Список редактируется в
настройках. Собственная доставка по России не должна получать агента.

## XML

Инструкция: [Как формировать чеки через XML](https://help.atol.online/article/11250).
Официальные [схемы и примеры](https://atol.online/upload/iblock/dd0/2z0p1h7itcce7yw9oxzsu0gax7idnrcq/xml_example.zip).

Загрузка выполняется в личном кабинете: `Загрузка чеков из файлов XML
(1.05/1.2)` -> компания -> магазин -> файл -> `Загрузить реестр (XML)`.

Для `<receipt>` порядок элементов должен соответствовать XSD. Тег 1192
сериализуется как `additional_check_props`. Для `<correction>` схема ФФД 1.05 не
разрешает `additional_check_props` и `additional_user_props`, поэтому генератор
их туда не добавляет.

## Отчёт АТОЛ

Вкладка `Журнал чеков` отдаёт CSV. По официальной инструкции текущий журнал
показывает последние две недели и выгружает до 10 000 строк; более старые данные
заказываются через `Архив чеков`.

Программа читает дату, тип, источник, статус, сумму, ФД, ФПД, ФН, UUID,
External Id, основание и JSON запроса/результата. При запуске она ищет последний
подходящий CSV в `atol_output/reports/atol` и в пользовательской папке загрузок.
