# Исправления по ФФД 1.05

Основной источник: [официальная инструкция АТОЛ](https://help.atol.online/article/11243).

## Главное различие

### Ошибочный чек уже существует

Для ФФД 1.05 формируются обычные чеки:

1. обычный чек с обратной операцией и ошибочными данными;
2. обычный чек исходной операции с правильными данными.

В обоих чеках в теге 1192 указывается ФПД первоначального ошибочного чека.
Например, ошибочный `sell` исправляется парой `sell_refund + sell`.

### Чек не был сформирован

Если расчёт был, дата прошла, а исходного фискального документа нет, формируется
один `sell_correction` или `buy_correction` с датой и номером основания.
Табличная часть для используемого проектом XML ФФД 1.05 не нужна. Тег 1192 в
элемент `<correction>` не добавляется, потому что это запрещено XSD АТОЛ.

Если ошибка обнаружена в тот же день и исходного чека нет, план может выбрать
обычный чек вместо коррекции.

## Сценарии программы

| Сценарий | Результат | Тег 1192 |
|---|---|---|
| Полная отмена лишнего чека | Один обычный обратный чек | Да |
| Чек большей суммой | Обратный чек + правильный обычный чек | В обоих |
| Чек меньшей суммой | Обратный чек + правильный обычный чек | В обоих |
| Неправильный способ оплаты | Обратный чек + правильный обычный чек | В обоих |
| Неправильная номенклатура | Обратный чек + правильный обычный чек | В обоих |
| Неправильная дата, исходный чек найден | Обратный чек + правильный обычный чек | В обоих |
| Чек не пробит и дата прошла | Один чек коррекции | Нет |
| Коррекция расхода без исходного чека | Один `buy_correction` | Нет |

Реальный возврат денег не является исправлением. Он удалён из сценариев Obsidian
и оформляется в разделе `Возвраты по заказам` как обычный `sell_refund` без тега
1192.

## Отдельное окно исправлений

После подготовки плана выбранные строки передаются в отдельное окно. В нём:

- нет команды отправки через API;
- видны все чеки плана и их порядок;
- отдельно редактируются ошибочная и правильная табличные части;
- отдельно выбираются ставки НДС исходного и правильного чеков, включая
  ставки 20%, 20/120, 22% и 22/122;
- при необходимости ставка меняется у отдельной позиции табличной части;
- отдельно задаются исходная и правильная оплаты;
- отображаются ФП исходного чека, основание, НДС и предупреждения;
- формируются XML и одна служебная записка на один исправляемый случай.

Для пары чеков создаётся одна служебная записка, описывающая оба документа.

## Обязательные проверки

Перед генерацией должны быть определены:

- сценарий;
- дата документа и правильная сумма из 1С;
- исходный чек строго по ФПД, если он существует;
- операция и сумма исходного чека;
- способ оплаты;
- вид услуги, агент и ставка НДС для услуг;
- исходная и правильная номенклатура для двухчекового исправления.

Неопределённая строка не попадает в XML и не мешает сформировать остальные.

## Служебная записка

Для каждого чека коррекции и каждого исправляемого случая хранится обоснование.
Официальная база АТОЛ также рекомендует сохранять заявление, акт или служебную
записку после массовой корректировки: [инструкция](https://help.atol.online/article/21114).
