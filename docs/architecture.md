# Архитектура и поток данных

## Технологии

- WPF, .NET 8, Windows;
- MVVM с `CommunityToolkit.Mvvm`;
- `ClosedXML` и Open XML для CSV/XLSX;
- `Microsoft.Web.WebView2` для кабинетов АТОЛ и Такском;
- COM-внешнее соединение `V83.COMConnector` для чтения 1С УТ 10.3;
- DOCX для служебных записок.

## Основные экраны

| Экран | Назначение |
|---|---|
| Возвраты по заказам | Разбор сообщения бухгалтера и `sell_refund` через API |
| Коррекции реализаций | Загрузка реализаций из 1С и подготовка XML |
| Исправление чеков | Синхронизация списка Obsidian, 1С и исходных чеков ОФД |
| Окно пробития исправлений | Отдельная рабочая форма XML-исправлений без API |
| Отчёты | История программы, журнал АТОЛ, ОФД/Такском и CSV для 1С |
| Оплаты / приход | Обычные чеки оплаты и редкие ручные операции |
| Настройки | Подключения, кассир, агенты, темы, Obsidian и версия |

## Поток обычного чека

```text
сообщение или ручной ввод
        -> OrderEntry
        -> проверка услуги, агента, НДС и оплаты
        -> API v4 АТОЛ Online
        -> UUID и опрос статуса
        -> ФД/ФПД в punched_checks.jsonl
```

## Поток коррекции

```text
1С или Obsidian
        -> данные документа и исходного чека ОФД
        -> план исправления ФФД 1.05
        -> отдельное окно исправлений
        -> XML + служебная записка
        -> ручная загрузка XML в АТОЛ Online
        -> журнал АТОЛ + отчёт ОФД
        -> CSV
        -> внешняя обработка 1С
```

Коррекции и Obsidian-исправления намеренно не отправляются кнопкой API. В
интерфейсе и команде выполнения действует двойная блокировка.

## Рабочие файлы

Файлы создаются рядом с запущенным `AtolGenerator.exe`.

| Файл или папка | Содержимое |
|---|---|
| `app_settings.json` | тема, кассиры, агенты, пути к отчётам и Obsidian |
| `atol_settings.json` | логин, пароль и group code АТОЛ |
| `onec_settings.json` | сервер, база, пользователь и пароль 1С |
| `obsidian_settings.json` | путь к рабочему Markdown-файлу |
| `atol_log.txt` | технический журнал API |
| `1c_log.txt` | журнал COM-запросов 1С |
| `punched_checks.jsonl` | локальная история успешно пробитых через API чеков |
| `atol_output/` | XML, DOCX, отчёты, кэш Такском и CSV для 1С |

Эти файлы не являются частью исходного кода и не должны коммититься.

## Границы ответственности

- Обычные реализации кассиры пробивают в 1С, а не в AtolGenerator.
- AtolGenerator читает 1С через COM, но не записывает фискальные поля напрямую.
- Изменения документов выполняет внешняя обработка в интерактивном сеансе 1С.
- ОФД подтверждает факт фискализации, но не заменяет данные исходного документа 1С.
- Obsidian хранит очередь и пояснения, но не является источником суммы или ФПД.
