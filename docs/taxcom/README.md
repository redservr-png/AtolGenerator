# Такском-Касса и отчёты ОФД

Такском используется как источник фактически фискализированных документов по
интернет-магазину и физическим кассам подразделений.

## Сводный отчёт

Официальная инструкция: [Формирование отчётов](https://taxcom.ru/support/reporting/).

Нужен `Сводный отчёт по фискальным документам` в XLSX. Программа читает:

- дату и время;
- документ и тип операции;
- признак расчёта;
- сумму;
- ФД и ФПД;
- номер ФН и РНМ ККТ;
- торговую точку/подразделение и название ККТ;
- ссылку на чек, если она присутствует.

Текущая официальная инструкция указывает предел 10 000 строк на один отчёт.
Большие периоды нужно разбивать или сужать фильтрами по торговой точке и ККТ.
Неполный отчёт нельзя трактовать как доказательство отсутствия чека.

При запуске программа ищет последний XLSX в `atol_output/reports/taxcom`, в
пользовательской папке загрузок и по последнему сохранённому пути. Несколько
файлов объединяются в архив с удалением полных дублей.

## Поиск конкретного чека

Официальная инструкция: [Поиск чека](https://taxcom.ru/support/poisk-cheka/).

Если в XLSX нет строки, программа может открыть встроенное окно WebView2:

1. пользователь авторизуется в Такском один раз;
2. для группы чеков задаётся общий период;
3. программа вводит ФПД и нажимает `Поиск чека`;
4. из карточки читаются дата, сумма, операция, ФД, ФПД, ФН и РНМ;
5. найденные данные сохраняются в
   `atol_output/reports/taxcom/online_receipts.json`;
6. результат применяется к выбранным строкам Obsidian.

Автоматизация зависит от DOM и поведения сайта Такском. После изменения
интерфейса селекторы нужно проверять. Внутренние HTTP-запросы сайта не считаются
публичным API-контрактом и в проекте не используются.

## Почему чек может не найтись в XLSX

- выбран не тот период;
- применён фильтр торговой точки, подразделения или ККТ;
- отчёт достиг ограничения строк;
- сформирован не сводный отчёт по фискальным документам;
- чек поступил после формирования файла;
- файл охватывает только часть касс.

Статус `Не найден` должен означать только результат поиска по загруженным данным.
Окончательное отсутствие подтверждается отдельным поиском по ФПД в кабинете.

## API

На дату 17.07.2026 в открытой официальной документации не обнаружена публичная
API-спецификация для экспорта чеков из кабинета Такском. До появления официального
договора API используются XLSX и управляемый браузер.
